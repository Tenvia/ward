# Ward v0.1.0-rc1 — tenant containment evaluation release

Pause one customer's AI agents. Everyone else keeps running.

## What Ward is

Ward is tenant containment for AI SaaS: an OpenAI-compatible egress
proxy plus a tenant control plane. Route agent/tool calls through Ward
with a tenant ID header; when one tenant's agent goes sideways, an
operator constrains or pauses that tenant — with an approval step and
an audit trail — while every other tenant keeps running.

## What this RC proves

- The core containment story, end to end and automated: Globex loops,
  Ward detects pressure, an operator approves and applies a
  constraint, Globex is blocked (429), Acme never blips (200), the
  audit trail records everything.
- Ward runs with zero NPM on the user's machine: one Docker container
  serving the API, the Control Room UI, and the OpenAPI contract on
  port 4317, with SQLite persistence across restarts.
- The live API conforms to the formal contract
  (`openapi/ward.v0.yaml`): 26 response-conformance checks run in the
  release battery.
- Reliability posture: SDK guard fail-open/fail-closed, proxy degraded
  fail-open with `x-ward-fail-open` evidence, and a fail-closed
  overlay — verified by smoke tests.

## Install (evaluation)

```bash
git clone https://github.com/Tenvia/ward && cd ward
docker compose -f docker-compose.pull.yml up
# open http://localhost:4317  (Control Room; demo token: ward-demo-token)
```

Or build locally: `docker compose -f docker-compose.user.yml up --build`,
then verify with `./scripts/smoke-user-install.sh` (docker + curl only).
Walkthrough: `docs/DESIGN_PARTNER_QUICKSTART.md`.

## Demo-supported in this RC

Egress proxy with per-tenant enforcement; pressure detection (flags,
never auto-constrains); approval-token flow with exact phrase echo;
audit trail; SQLite persistence (incl. across container restarts);
bundled Control Room (browser-tested, with and without control auth);
no-NPM user path; served OpenAPI contract + live conformance; uv/uvx
`wardctl`; dev-only Docker runner behind a flag with image allowlist.

## Not production-ready — do not deploy for real traffic

- Single node only; no HA; hard-down of the Ward process stops proxied
  traffic (customer-side fallback routing is your responsibility).
- Control auth is one shared bearer token (prototype, not RBAC).
- SQLite persistence is a prototype (JSON blobs, experimental
  `node:sqlite`); approval tokens and workflow runs are in-memory.
- Cost figures are fixed per-call estimates, not metering.
- The OpenAI-compatible endpoint is demo-grade, not a full spec clone
  (no streaming); pass-through upstream mode exists but is UNTESTED.
- Kubernetes runner: planned. Hosted Ward: planned.
- License: Apache-2.0 (`LICENSE`, SPDX `Apache-2.0`), added to the
  repo on 2026-07-05 shortly after the rc1 image publish — the rc1
  image itself predates it, so its `org.opencontainers.image.licenses`
  label is empty; the next published tag will carry the label.
